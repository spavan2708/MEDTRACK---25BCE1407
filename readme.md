**DA1 · Question 8 · C Programming + WebAssembly**

## Project Overview

MedTrack is a menu-driven hospital patient management system built in C, compiled to WebAssembly (WASM) and hosted on GitHub Pages. It allows clinic staff to register patients, record visit diagnoses, search records, and identify frequent visitors — all running live in the browser.

---

## C Concepts Used

| # | Concept | Implementation |
|---|---------|----------------|
| 1 | struct | `Patient`, `Visit` (typedef structs) |
| 2 | File Handling | `fopen`, `fread`, `fwrite`, `fclose` — binary `.dat` files |
| 3 | User-defined Functions | 17 functions including `addPatient`, `addVisit`, `editPatient`, `deletePatient`, `searchPatient`, `validatePhone`, `validateAge`, `getFrequentVisitors` and more |
| 4 | Menu-driven Program | `main()` with `do-while` loop + `switch/case` |
| 5 | Input Validation | Phone: exactly 10 digits · Age: 0–150 · Duplicate ID check |
| 6 | String Operations | `strcmp`, `strcpy`, `strncpy`, `strstr`, `strlen`, `snprintf` |
| 7 | Arrays & Pointers | `patients[]`, `visits[]`, `char*` return values |

---

## Features

- Register patient (ID, Name, Age, Phone, Gender, Blood Group, Address)
- Add visit record (Date, Diagnosis, Prescription, Doctor, Severity)
- Search patient by ID, Name, or Phone
- View full visit history per patient
- List all registered patients
- Identify frequent visitors (3 or more visits)
- System statistics with severity breakdown
- Edit patient record
- Delete patient and all associated visits
- Save and load data via binary `.dat` files (C) / `localStorage` (browser)

## How to Compile to WebAssembly

**One-time Emscripten setup:**
```bash
cd ~
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
./emsdk install latest
./emsdk activate latest
source ./emsdk_env.sh
```

**Every new terminal session:**
```bash
source ~/emsdk/emsdk_env.sh
```

**Compile:**
```bash
cd ~/Downloads/UM
emcc main.c -o index.js \
  -s EXPORTED_FUNCTIONS='["_main","_addPatient","_addVisit","_editPatient","_deletePatient","_searchPatient","_getAllPatients","_getAllVisits","_getVisitHistory","_getFrequentVisitors","_getStats"]' \
  -s EXPORTED_RUNTIME_METHODS='["ccall"]' \
  -s ALLOW_MEMORY_GROWTH=1 \
  -s NO_EXIT_RUNTIME=1
```

This generates `index.js` and `index.wasm` in your folder.

## How to Host on GitHub Pages

1. Create a new **public** repository on [github.com](https://github.com)
2. Upload `index.html`, `index.js`, `index.wasm` (and optionally `main.c`)
3. Go to **Settings → Pages → Branch: main / root → Save**
4. Your live link will be: `https://yourusername.github.io/repo-name/`

## Input Validation Rules

- **Patient ID** — required, must be unique
- **Age** — must be a number between 0 and 150
- **Phone** — must be exactly 10 digits, no spaces
- **Date** — must follow YYYY-MM-DD format
- **Severity** — one of: Routine, Moderate, Severe, Emergency

## Common Issues & Fixes

| Problem | Fix |
|---------|-----|
| `emcc not found` | Run `source ~/emsdk/emsdk_env.sh` again |
| GitHub Pages shows 404 | Wait 2–5 minutes and refresh. Ensure repo is public and branch is main/root |
| Browser blocks WASM | Must serve via `http://` — never open `index.html` directly as a file |
| `gcc` not found on Mac | Run `xcode-select --install` in Terminal |
